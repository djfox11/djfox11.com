Super Mario Odyssey Model Ripping Tutorial

Learn how to extract, convert, and prepare Super Mario Odyssey models for Blender and submission.

Tutorial last updated: 28/05/2023

This guide walks through extracting models from Super Mario Odyssey, converting them with Switch Toolbox, setting up texture maps in Blender, and preparing the finished model for submission.

Important: This tutorial assumes you already have a legally obtained dump of the game and that Yuzu or Ryujinx is already installed and configured. It does not include instructions for obtaining game files or setting up emulators.

You will learn

  • How to extract the base game files
  • How to import and convert ripped models
  • How to set up texture maps in Blender
  • How to prepare models for upload

Requirements

Game files

  • A legally obtained NSP or XCI dump of Super Mario Odyssey
  • NSP: Nintendo Submission Package
  • XCI: Game Cartridge Image

Software

  • Yuzu or Ryujinx
  • Switch Toolbox
  • Blender, or another modelling program of your choice
  • Model Checker
Note: Model Checker can help identify common issues with a model, but it does not guarantee that a submission will be approved.

Extracting the game

For Yuzu

  1. Make sure Super Mario Odyssey appears in the games list.
  2. Right-click the game and choose Dump RomFS > Dump RomFS.
  3. When asked how you would like the game dumped, select Full.
  4. Wait for the extraction to finish.
  5. When File Explorer opens, navigate to the ObjectData folder.
  6. Keep note of this directory, as you will need it later.

For Ryujinx

  1. Make sure Super Mario Odyssey appears in the games list.
  2. Right-click the game and choose Extract Data > RomFS.
  3. Choose a location for the dump.
  4. Keep note of the dump directory for later.

Extracting and converting the models

This section uses Switch Toolbox to open the extracted game files, locate the model packages, and export the model as a Collada file.

  1. Extract the downloaded ZIP archive named Toolbox-Latest.zip.
  2. Inside the extracted folder, open Toolbox.exe.
  3. In Switch Toolbox, go to File > Open > Folder.
  4. Navigate to the extracted Super Mario Odyssey RomFS folder, select the ObjectData folder, then click Select Folder.
  5. You should now see a long list of .szs files. These packages can contain models, textures, animations, and other files.
  6. Find the model you want to extract and double-click its corresponding .szs file.
  7. Inside the package, you should see a .bfres file and some .byml files. The .bfres file is the one you want.
  8. Double-click the .bfres file to open the model in the viewport.
  9. Expand the Models folder under the .bfres.
  10. Right-click the model, choose Extract, and export it as a .dae file.
Tip: If you only want to use the model in another project, you can stop after exporting it. If you are creating submission icons, continue into Blender.

Importing the model into Blender

  1. Open Blender.
  2. Go to File > Import > Collada (.dae).
  3. Select the model you exported earlier.
  4. Once imported, the model will probably be very large.
  5. Select the armature in the Outliner and scale the model down to a reasonable size. I usually scale it to 0.01.
Submission note: Only follow the material setup steps if you are creating icons or preview renders. Do not export the final submission model with these temporary texture map adjustments applied.

Setting up texture maps

After importing the model, switch to Blender’s Shading workspace. This is where the exported textures can be assigned and previewed.

  1. Add a new Image Texture node and select the grey roughness texture.
  2. Plug the Color output of the roughness texture into the Roughness input of the Principled BSDF node.
  3. Change the roughness texture’s Color Space to Non-Color.
  4. If metallic or emission textures are present, repeat the same general process for those maps.
  5. Add another Image Texture node and select the purple normal map.
  6. Add a Normal Map node.
  7. Plug the normal map texture’s Color output into the Color input of the Normal Map node.
  8. Plug the Normal output of the Normal Map node into the Normal input of the Principled BSDF.

Preparing the model for submission

  1. Once the model has been scaled to a reasonable size, go to File > Export > Collada (.dae).
  2. Leave the export settings as they are and click Export.
  3. Open the exported .dae file in Model Checker.
  4. If everything has gone well, Model Checker should report no major issues.
  5. If any issues appear, Model Checker should identify them and explain what needs fixing.

FAQ

When opening the model’s .bfres, there are no textures. Where are they?

If there are no textures visible on the loaded model in Switch Toolbox, don’t worry. The textures are most likely stored in another .szs file. For example, kingdom-related models often have separate texture packages with names ending in StageTexture.

You usually do not need to manually find and extract these texture packages. Switch Toolbox should find and extract the required textures automatically.

Final words

That’s it! You should now have a Super Mario Odyssey model extracted, converted, checked, and ready to work with.

If something goes wrong, check the troubleshooting section, double-check your file paths, and make sure the model was exported from the correct .bfres file.
